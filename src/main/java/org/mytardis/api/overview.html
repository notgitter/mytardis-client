This package implements a Java client for the MyTardis API. It contains two sub-packages:

org.mytardis.api.client

Includes the classes for building, posting, and retrieving objects from MyTardis.

Experiments are built using the {@link org.mytardis.api.client.ExperimentTree}, {@link org.mytardis.api.client.DatasetTree}, and {@link org.mytardis.api.client.DatafileTree} classes---each of which can have associated meta-data parameters. Trees take a {@link org.mytardis.api.client.TardisClient} instance as a constructor parameter---which handles the interaction with the MyTardis RESTful interface.

Trees can be validated individually using an associated checkTree() method. Alternatively, the {@link org.mytardis.api.client.ExperimentTree#verify} method can be used to execute all checkTree() methods in an experiment and aggregate any errors that they generate. This verify() method must validate successfully---with no errors---for the {@link org.mytardis.api.client.ExperimentTree#post} method to push the experiment, datasets, and datafiles to the MyTardis system. See {@link org.mytardis.api.client.ExperimentTree#verify} for details of which attributes are required and which are optional for each tree.

A {@link org.mytardis.api.client.TardisClient} can be used to retrieve {@link org.mytardis.api.client.TardisObject} instances for a given model, either as a list or individually---by Id or ResourceURI.

org.mytardis.api.model

Includes the object models generated from the MyTardis API schema. Objects are generated using the jsonschema2pojo utility, using the following parameters: source type is "JSON", and annotation style is "Gson".
